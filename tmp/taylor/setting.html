David’s Factoring Problem

A well known NP-intermediate problem, integer factoring, serve as an important role in many RSA crypto-systems. David, as a number theorist, created another kind of factoring as following.

Given 𝑎0,𝑎1∈{0,1,…𝑝−1}a0,a1∈{0,1,…p−1} and 2≤𝑝<2312≤p<231 a prime. Find 𝑏0,𝑏1∈{0,1,…𝑝−1}b0,b1∈{0,1,…p−1} so the following holds,

𝑎0=𝑏0⋅𝑏1mod𝑝𝑎1=𝑏0+𝑏1mod𝑝.a0=b0⋅b1modpa1=b0+b1modp.

“This factoring is way more cool, in the sense that it can now be computed efficiently,” said David. Now, he invites you to enjoy this new variant of factoring.

Input

The first line contains an integer 1≤𝑇≤1001≤T≤100 indicating the number of test cases.
For each test cases, there’s a line of 𝑝,𝑎0,𝑎1p,a0,a1 seperated by a single blank.

Output

For each test case, output a line containing 𝑏0,𝑏1b0,b1 in acending order seperated by a single blank. If there’s no solution, output a line of −1−1.

Sample

  • Input/Output 1
2
2 1 0
2 1 1

1 1
-1
